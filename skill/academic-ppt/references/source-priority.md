# Source Priority

## Principle

Not all files should be treated as equally authoritative.

Prefer sources that best represent the final thesis argument and the student's actual work.

## Default Precedence

1. Explicit user-designated canonical source
2. Final thesis manuscript
3. Latest thesis draft
4. Project report or paper manuscript by the same student
5. Existing presentation deck
6. Standalone notes or markdown summaries
7. Images and figures as supporting evidence

## File-Type Heuristics

- `.docx` and `.pdf` thesis files usually dominate narrative extraction
- `.md` files often contain newer summaries or hand-edited wording
- `.pptx` files are usually reference structure or prior deck content, not the full truth source
- images support explanation but should not define the main narrative alone

## Conflict Resolution

When sources disagree:
- prefer explicit user instruction over file heuristics
- prefer later-stage thesis material over earlier-stage notes
- prefer thesis conclusions backed by methods/results over isolated summary statements
- if uncertainty remains material, ask the user

## Attribution Rules

- Track which source each slide mainly came from
- Keep notes on borrowed figures, third-party visuals, and reused text
- Distinguish student contribution from cited or inherited material
